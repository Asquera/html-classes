Lesson 1: Familiarize yourself with basic HTML

Get your tools ready

To follow this class, you need at least these tools:

  • A text editor (TextMate and Coda are both good options)
  • Your preferred Browser, we recommend Safari, Firefox and Google Chrome
  • A Textbook to take notes and make little sketches of what you want to implement.

Later on, we will learn about other tools like the W3C Validator and your preferred browsers developer support, so go ahead and have a look at them if you are interested.

Terminology

Source
The HTML document as displayed in your text editor.
Page
The page displayed in the browser.

Exercise 1.1: Reading HTML

Have a look at this document itself, both in a browser and a text editor. Try to find out how each of the parts of the html documents gets displayed. Try to figure out their meaning.
Bonus: Try to classify the elements in groups.

Turn all this into a small table[1]:

Element Description Groups
<meta> The meta tag allows you to describe meta information of the document. only in head, not displayed

While doing that, answer the following questions:

  1. How do line breaks in the source display in the browser?
  2. Try to find relationships between the tag names and their display properties.
  3. Most tag names are abbreviations. What is their meaning?
  4. What does the DOCTYPE statement mean and why is it important?

Exercise 1.2: Create a Boilerplate

Create a small page that that serves as a starting point for the following exercises. Put it in a project folder and name it boilerplate.html. We will evolve this boilerplate with new stuff. Feel free to personalize it, as long as it doesn't keep you from getting started immediately.

For now, your Boilerplate should contain:

  1. The DOCTYPE statement
  2. A <title>-element
  3. A heading
  4. A paragraph
  5. Some kind of listing

It should also be correct. Check with the W3C Validator, if in doubt.
